History Table Rendering - Before & After Refactoring

Changes Made:

  • Created helper function formatHistoryComment() for smart JSON rendering
  • Full snapshots now show "Details aktualisiert" instead of all fields
  • Change data shows field-by-field changes in "Field: Old → New" format
  • Uses HTML definition lists (<dl>) for structured data
  • Robust JSON error checking with fallback to plain text
  • Filters out empty values and unwanted fields (image_path, original_data)

Summary of Improvements

  • Better readability: Change data formatted as "Old → New" instead of raw JSON
  • Cleaner interface: Full snapshots show summary instead of all fields
  • Semantic HTML: Uses proper <dl> tags for definition lists
  • Smart filtering: Removes unwanted fields (image_path) and empty values
  • Robust error handling: Gracefully handles invalid JSON
  • Maintained functionality: Plain text comments work exactly as before
