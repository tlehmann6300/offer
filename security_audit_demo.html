Über diese Demo

Diese Seite demonstriert die Security Audit Warnung, die automatisch im Admin-Dashboard erscheint, wenn sensible Installationsdateien auf dem Server gefunden werden.

Die Warnung wird nur für Benutzer mit admin Berechtigung angezeigt und prüft automatisch bei jedem Dashboard-Aufruf, ob kritische Dateien noch vorhanden sind.

Geprüfte Dateien:

  • • setup_admin.php - Admin-Setup-Skript
  • • create_admin.php - Admin-Erstellungsskript
  • • cleanup_*.php - Cleanup-Skripte
  • • sql/migrations/ - SQL-Migrations-Verzeichnis
  • • Weitere Debug- und Wartungsskripte

Tipp: Im Produktivbetrieb sollten alle diese Dateien entfernt werden, um unbefugten Zugriff zu verhindern.

Integration Code 

<?php
// In pages/dashboard/index.php

// Security Audit - nur für Admins
if (AuthHandler::hasPermission('admin')) {
    require_once __DIR__ . '/../../security_audit.php';
    $securityWarning = SecurityAudit::getDashboardWarning(__DIR__ . '/../..');
}

// Im Template ausgeben
if (!empty($securityWarning)) {
    echo $securityWarning;
}
?>
